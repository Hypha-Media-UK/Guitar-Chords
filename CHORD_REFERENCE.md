## Data Structure
- Array format: `[String 1 (high E), String 2 (B), String 3 (G), String 4 (D), String 5 (A), String 6 (low E)]`
- Index 0 = String 1 (high E)
- Index 5 = String 6 (low E)
- This is the OPPOSITE of standard chord charts which show low E to high E
- When converting from standard notation, the array must be REVERSED

## Verification Methodology

Each chord was verified against:
1. **Standard guitar chord theory** (intervals, voicings, note relationships)
2. **Common fingering patterns** used by professional guitarists
3. **Playability** (ensuring fingerings are physically possible and comfortable)
4. **Barre chord conventions** (same finger number for barred strings at same fret)

## Notes
- Barre chords use the same finger across multiple strings at the same fret
- 'x' means don't play that string
- 'o' or '0' means play the string open
- Numbers 1-4 represent fingers: 1=index, 2=middle, 3=ring, 4=pinky
